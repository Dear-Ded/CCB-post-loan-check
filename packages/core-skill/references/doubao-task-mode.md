# 豆包任务模式适配设计

豆包任务模式可视为带浏览器能力的任务执行环境。适配重点是稳定状态机，而不是固定本机路径。

## 状态机

1. 收集企业信息。
2. 启动浏览器会话。
3. 打开需要人工登录或页面确认的来源。
4. 一次性提示用户完成必要登录或确认项。
5. 等待页面进入可查询状态。
6. 自动填充企业名、统一社会信用代码或组织机构代码或个人身份证号。
7. 点击查询。
8. 如果页面确认失败、页面无响应或查询页加载失败，则保持当前页并等待用户重新确认。
9. 只在确认查询结果/无结果状态后截图。
10. 汇总截图并生成报告。

## 截图验收

执行信息公开网截图必须满足：

- URL 属于 `zxgk.court.gov.cn`。
- 页面不是登录页、挑战失败页、异常页或空白页。
- 页面文本出现查询结果、未查询到、暂无数据、执行法院、案号、立案时间、执行标的等结果态信号之一。

不满足时不得进入最终 Word。
